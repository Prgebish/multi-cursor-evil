EVM Tutorial 16: Restrict Search Region
==========================================

By default, C-n and n search the entire buffer. The
restrict command limits the search to a specific region,
so matches outside that region are ignored.

  \ r  = toggle restriction (set or clear)

Set: select a region in visual mode, then press \ r.
Clear: press \ r while evm is active with a restriction.

The leader key (\ by default) is set by evm-leader-key.
The restricted area is highlighted with an overlay.


Lesson 1: Restrict Before Search
-----------------------------------

Select lines with V (visual line mode), press \ r to
set a pending restriction. Then C-n starts a search
that only finds matches within the restricted area.

> Try it: change "old" only in the first block.
  1. Place cursor on "name" on line 1 of the exercise below
  2. Press V then jj — visual select lines 1-3
  3. Press \ r — restriction set (area highlighted)
  4. Place cursor on "old" on any line within the block
  5. Press C-n, then n twice (3 matches, only in block 1)
  6. Press c, type new, press Esc — all three changed
  7. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  name = old
  value = old
  count = old

  name = old
  value = old
  count = old
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result (only first block changed):
  name = new
  value = new
  count = new

  name = old
  value = old
  count = old

  The second block is untouched — its matches were
  outside the restriction.


Lesson 2: Clear Restriction (\ r)
------------------------------------

Press \ r again (without a visual selection) while evm
is active to remove the restriction. After clearing,
n finds matches in the entire buffer.

> Try it: restrict, search, then clear and find more.
  1. Place cursor on "x" on line 1 below
  2. Press V then j — visual select lines 1-2
  3. Press \ r — restriction set
  4. Place cursor on "x" on line 1, press C-n
  5. Press n — finds 1 more "x" (line 2, within restriction)
     Lines 3-4 are skipped.
  6. Press \ r — restriction cleared (highlight removed)
  7. Press n — now finds "x" on line 3 too
  8. Press Esc when done

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  x = 1
  x = 2
  x = 3
  x = 4
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  With restriction: only 2 matches (lines 1-2).
  After clearing: n continues to find the rest.


Lesson 3: Practical Workflow
-------------------------------

Rename a variable in one function without affecting
the rest of the file.

> Try it: rename "data" to "result" in process() only.
  1. Place cursor on "data" in "data = fetch()" below
  2. Press Vjj — visual select the 3 lines of process()
  3. Press \ r — restriction set
  4. Place cursor on "data" on any line in the block
  5. Press C-n, then n twice (3 matches in process only)
  6. Press c, type result, press Esc — renamed
  7. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  def process():
      data = fetch()
      print(data)
      return data

  def backup():
      data = read_all()
      print(data)
      return data
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  def process():
      result = fetch()
      print(result)
      return result

  def backup():
      data = read_all()
      print(data)
      return data

  Only process() was affected. The restriction kept
  backup() completely untouched.

  Quick reference:
    \ r in visual mode  = set restriction
    \ r in evm mode     = clear restriction
    Restriction limits C-n, n, and N searches
    The restricted area is visually highlighted
