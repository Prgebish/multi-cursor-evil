EVM Tutorial 14: Align Cursors
===============================

The align command inserts spaces before each cursor/region
so that all cursors line up at the same column — the column
of the rightmost cursor.

  \ a  = align all cursors/regions vertically

Works in both cursor and extend mode.
The leader key (\ by default) is set by evm-leader-key.


Lesson 1: Align in Extend Mode
---------------------------------

When you search with C-n, matches often land at different
columns. Align inserts spaces before each selection to line
them all up at the rightmost one.

> Try it: align the values across lines.
  1. Place cursor on "Alice" on line 1 below, press C-n
  2. Press n twice (3 matches of "Alice")
  3. Press \ a — spaces inserted to align selections
  4. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  x = Alice
  long_name = Alice
  y = Alice
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  x =         Alice
  long_name = Alice
  y =         Alice

  Line 2 was already at the rightmost column (12), so
  only lines 1 and 3 got extra spaces.


Lesson 2: Align with f-Motion
--------------------------------

Use C-Down to create cursors, then f (find char) to jump
each cursor to a target character. They land at different
columns — then align lines them up.

> Try it: align the equal signs.
  1. Place cursor on "a" on line 1 below
  2. Press C-Down twice (3 cursors)
  3. Press f= — each cursor jumps to "=" on its line
  4. Press \ a — equal signs aligned
  5. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  a = 1
  bb = 2
  ccc = 3
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  a   = 1
  bb  = 2
  ccc = 3

  After f=, cursors land on "=" at different columns.
  Align adds spaces to bring all to the rightmost one.


Lesson 3: Align with w-Motion
--------------------------------

w jumps to the start of the next word. When first words
differ in length, cursors end up at different columns —
perfect for align.

> Try it: align the types to a uniform column.
  1. Place cursor on "n" of "name" on line 1 below
  2. Press C-Down twice (3 cursors)
  3. Press w — each cursor jumps to the second word
  4. Press \ a — types aligned
  5. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  name string
  age int
  active bool
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  name   string
  age    int
  active bool

  After w, cursors land on types at different columns.
  Align pushes all to the rightmost one ("bool").


Lesson 4: Practical Workflow
-------------------------------

Combine align with editing: align first, then modify
the text — all in one multi-cursor session.

> Try it: align equal signs and change them to colons.
  1. Place cursor on "a" on line 1 below
  2. Press C-Down twice (3 cursors)
  3. Press f= — each cursor jumps to "="
  4. Press \ a — equal signs aligned
  5. Press r: — replace "=" with ":" at all cursors
  6. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  a = 10
  bb = 200
  ccc = 3
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  a   : 10
  bb  : 200
  ccc : 3

  Quick reference:
    \ a = align cursors/regions to rightmost column
    Works in both cursor and extend mode
    Inserts spaces before each cursor/region start
