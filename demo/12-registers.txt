EVM Tutorial 12: VM Registers
===============================

EVM maintains its own register system. When you yank (y) or
delete (d) with multiple cursors, each cursor's text is stored
as a separate entry in the register — not concatenated.

This enables the key feature: pasting distributes entries
1-to-1 back to cursors.

  3 cursors yank "a", "b", "c"  ->  register = ["a","b","c"]
  3 cursors paste                ->  cursor 1 gets "a",
                                     cursor 2 gets "b",
                                     cursor 3 gets "c"


Lesson 1: Yank and Paste in Extend Mode (y, p)
-------------------------------------------------

y in extend mode copies each selection into the default
register. p pastes — replacing the current selections if
in extend mode, or inserting if in cursor mode.

> Try it: yank the keywords, then replace the placeholders.
  1. Place cursor on "A" of "Alice" on line 1 below
  2. Press C-Down twice (3 cursors on Alice, Bob, Carol)
  3. Press yiw — yanks ["Alice", "Bob", "Carol"]
     (message: "Yanked 3 regions")
  4. Press Esc to exit evm
  5. Place cursor on "___" on line 5, press C-n
  6. Press n twice (3 matches of "___" selected)
  7. Press p — each "___" replaced with its value
  8. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Alice
  Bob
  Carol

  Hello, ___!
  Hello, ___!
  Hello, ___!
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  Alice
  Bob
  Carol

  Hello, Alice!
  Hello, Bob!
  Hello, Carol!

  The register had 3 entries and there were 3 selections,
  so each got its matching value (1-to-1 distribution).


Lesson 2: Cursor Mode Paste (p, P)
-------------------------------------

In cursor mode, p inserts AFTER the cursor (like vim's p).
P inserts BEFORE the cursor (like vim's P).

> Try it: yank the keys and paste them after the colon.
  1. Place cursor on "a" on line 1 below
  2. Press C-Down twice (3 cursors)
  3. Press yiw — yanks "a", "b", "c" (inner word)
  4. Press $ — jump to end of each line (":")
  5. Press p — pastes after ":" at each cursor
  6. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  a:
  b:
  c:
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  a:a
  b:b
  c:c

  Each cursor's yanked word is distributed 1-to-1.
  Cursor 1 yanked "a" and gets "a" pasted, etc.


Lesson 3: Named Registers ("a prefix)
----------------------------------------

The " prefix lets you specify a named register (a-z),
just like in vim. This keeps multiple yanked sets separate.

  "ay = yank to register a
  "ap = paste from register a
  "by = yank to register b

> Try it: yank to a named register.
  1. Place cursor on "red" on line 1 below, press C-n
  2. Press n twice (3 matches of "red")
  3. Press "ay — yanks all to register 'a'
     (message: "Yanked 3 regions to register 'a'")
  4. Press Esc to exit evm

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  color = red
  color = red
  color = red
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Register 'a' now holds ["red", "red", "red"].
  You can yank something else to the default register
  and still paste from 'a' later with "ap.


Lesson 4: Broadcast — One Value to Many
------------------------------------------

When the register has fewer entries than cursors, values
are reused. The most common case: 1 value pasted to all
cursors (broadcast).

This happens naturally when you yank one word in normal
vim (outside evm) and then paste inside evm.

> Try it: paste the same value into all placeholders.
  1. Place cursor on "new" on line 1 below
  2. Press C-n (1 match, enter evm), then y (yank "new")
  3. Press Esc to exit evm
  4. Place cursor on "old" on line 3, press C-n
  5. Press n twice (3 matches of "old")
  6. Press p — all three get "new"
  7. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  new

  value = old
  value = old
  value = old
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  new

  value = new
  value = new
  value = new


Lesson 5: Uppercase Registers — Append ("A)
----------------------------------------------

Uppercase register names append to the existing content
instead of replacing it — same convention as vim.

  "ay = yank to register a (replace)
  "Ay = yank to register a (append)

This is useful for collecting text from several selections
into one register across multiple evm sessions.


Lesson 6: Practical Workflow
-------------------------------

A real scenario: extract values from one structure and
insert them into another.

> Try it: copy the sizes into the function call.
  1. Place cursor on "1" of "10" on line 1 below
  2. Press C-Down twice (3 cursors on "10", "20", "30")
  3. Press yiw — yanks ["10", "20", "30"]
  4. Press Esc to exit evm
  5. Place cursor on "0" on the last line, press C-n
  6. Press n twice (3 matches of "0" in template)
  7. Press p — replaces "0" with "10", "20", "30"
  8. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  width  = 10
  height = 20
  depth  = 30

  size(0, 0, 0)
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  width  = 10
  height = 20
  depth  = 30

  size(10, 20, 30)

  Quick reference:
    y   = yank to default register    "ay = yank to 'a'
    p   = paste after    P = paste before    "ap = from 'a'
    "Ay = append to register 'a'

    N values -> N cursors = 1-to-1 distribution
    1 value  -> N cursors = broadcast (same everywhere)
