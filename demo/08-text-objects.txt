EVM Tutorial 08: Text Objects
==============================

Text objects are the most powerful editing primitives in vim.
Instead of motions (dw = delete from cursor forward to next word),
text objects select a logical unit around the cursor.

  i = inner — content only, without delimiters
  a = around — content + delimiters (or word + surrounding space)

Operator + text object: diw, ci", da(, yit, etc.
This works at ALL cursors simultaneously.


Lesson 1: Inner Word vs A Word (diw / daw)
--------------------------------------------

diw deletes the word under the cursor (no surrounding whitespace).
daw deletes the word plus one side of surrounding whitespace.

diw is great when you want to keep spacing intact.
daw is great when you want to remove the word cleanly from a list.

> Try it: delete the middle word with diw.
  1. Place cursor on "old" on line 1 below
  2. Press C-Down twice (3 cursors)
  3. Press diw — deletes "old" but keeps both spaces
  4. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  my old car
  my old bag
  my old hat
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result (double space remains):
  my  car
  my  bag
  my  hat

> Now try daw on the same text (undo first with u):
  1. Place cursor on "old" on line 1
  2. Press C-Down twice (3 cursors)
  3. Press daw — deletes "old" and the adjacent space
  4. Press Esc when done (u to undo)

  Expected result (clean single space):
  my car
  my bag
  my hat


Lesson 2: Change Inside Quotes (ci" / ci')
--------------------------------------------

ci" deletes the text inside double quotes and enters insert mode.
This is one of the most commonly used text objects in vim.
The cursor can be anywhere inside the quotes — it finds them
automatically.

> Try it: change the quoted strings.
  1. Place cursor anywhere inside "red" on line 1 below
  2. Press C-Down twice (3 cursors)
  3. Press ci" — the text inside quotes is deleted, insert mode
  4. Type "blue"
  5. Press Esc, then Esc

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  color = "red"
  color = "green"
  color = "yellow"
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  color = "blue"
  color = "blue"
  color = "blue"


Lesson 3: Parentheses (di( / da( / ci()
------------------------------------------

di( (or di) or dib) deletes the text inside parentheses.
da( deletes the text including the parentheses themselves.
ci( changes the text inside parentheses.

> Try it: change the function arguments.
  1. Place cursor on "x" on line 1 below (inside the parens)
  2. Press C-Down twice (3 cursors)
  3. Press ci( — text inside () is deleted, insert mode
  4. Type "result"
  5. Press Esc, then Esc

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  print(x + 1)
  print(y + 2)
  print(z + 3)
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  print(result)
  print(result)
  print(result)


Lesson 4: Brackets and Braces (di[ / di{)
-------------------------------------------

di[ deletes text inside square brackets.
di{ (or diB) deletes text inside curly braces.
Same pattern — i = inner, a = around.

> Try it: delete the array contents.
  1. Place cursor on "1" on line 1 below (inside the brackets)
  2. Press C-Down twice (3 cursors)
  3. Press di[ — text inside [] is deleted
  4. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  x = [1, 2, 3]
  y = [4, 5, 6]
  z = [7, 8, 9]
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  x = []
  y = []
  z = []

> Now try da[ to remove the brackets entirely (undo first):
  1. Same setup — cursor inside brackets, 3 cursors
  2. Press da[ — deletes [] and contents
  3. Press Esc when done (u to undo)

  Expected result (trailing space remains after "="):
  x =
  y =
  z =


Lesson 5: Tags (dit / dat)
-----------------------------

dit deletes the content inside an HTML/XML tag pair.
dat deletes the tags themselves along with their content.
cit changes the content inside a tag — perfect for HTML editing.

> Try it: change the tag content.
  1. Place cursor on "H" of "Hello" on line 1 below
  2. Press C-Down twice (3 cursors)
  3. Press cit — text inside tags is deleted, insert mode
  4. Type "New"
  5. Press Esc, then Esc

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  <li>Hello</li>
  <li>World</li>
  <li>Items</li>
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  <li>New</li>
  <li>New</li>
  <li>New</li>


Lesson 6: Combining All Operators with Text Objects
------------------------------------------------------

Summary of operator + text object combinations:

  d + iw  = delete inner word         d + aw  = delete a word
  c + i"  = change inside quotes      c + a"  = change around quotes
  y + i(  = yank inside parens        y + a(  = yank around parens
  d + it  = delete inside tags        d + at  = delete tag and content
  c + i[  = change inside brackets    c + a{  = change around braces

> Try it: yank inside quotes, then paste after.
  1. Place cursor inside "alpha" on line 1 below
  2. Press C-Down twice (3 cursors)
  3. Press yi" — like di" but copies instead of deleting
     (message: "Yanked 3 regions")
  4. Press $ to jump to end of each line
  5. Press p — pastes the yanked text after the cursor
  6. Press Esc when done (u to undo)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  key = "alpha"
  key = "bravo"
  key = "gamma"
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  key = "alpha"alpha
  key = "bravo"bravo
  key = "gamma"gamma


Lesson 7: Practical Workflow
-------------------------------

A real scenario: you have function calls with wrong arguments
and want to replace them using text objects.

> Try it: change the arguments in all three calls.
  1. Place cursor on "u" of "user_id" on line 1 below
  2. Press C-Down twice (3 cursors)
  3. Press ci( — content inside parens is deleted, insert mode
  4. Type "data"
  5. Press Esc (exit insert), then Esc (exit evm)

  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  validate(user_id, "admin")
  validate(user_id, "editor")
  validate(user_id, "viewer")
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

  Expected result:
  validate(data)
  validate(data)
  validate(data)

  Tip: text objects are especially powerful with multi-cursors
  because they don't require precise cursor placement. As long
  as each cursor is somewhere inside the delimiters, ci( / ci"
  will find the correct boundaries automatically.
